# 调研与决策：网络代理模块

## 决策 1：代理配置生效位置
- 决策：在 Python requests Session 中统一注入代理配置
- 理由：出站请求集中在 Python 侧，便于统一控制与关闭
- 备选：在每个请求点单独配置代理

## 决策 2：代理开关关闭策略
- 决策：关闭代理时禁用环境变量代理
- 理由：避免系统/环境变量代理绕过配置，确保行为可控
- 备选：仅设置空代理字典

## 决策 3：SOCKS 支持方式
- 决策：引入 PySocks 作为 requests 的 socks 支撑
- 理由：requests 原生不含 socks，需要额外依赖
- 备选：使用其他 HTTP 客户端替换 requests

## 决策 4：UI 放置位置
- 决策：在现有“网络代理”页新增配置面板
- 理由：符合用户心智与现有导航结构，保持风格一致
- 备选：在全局设置页新增独立入口
