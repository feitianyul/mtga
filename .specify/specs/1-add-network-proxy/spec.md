# 功能规范：添加网络代理模块

## 概述
在现有版本上新增网络代理功能，用于统一管理出站网络访问的代理设置，同时保持现有功能与界面风格不变。

## 背景与问题
部分用户在受限网络环境下需要通过代理访问外部服务，目前缺少可配置的网络代理入口，导致连接不稳定或不可用。

## 目标
- 提供全局网络代理开关
- 支持 HTTP、HTTPS、SOCKS4、SOCKS5 代理
- 支持可选的代理用户名与密码
- 代理配置在应用重启后自动读取并生效
- UI 风格与现有保持一致

## 非目标
- 不引入新的业务流程或改变已有功能入口
- 不新增额外的配置文件作为代理配置来源
- 不改变现有日志输出与功能行为的语义

## 假设与约束
- 代理配置必须写入现有用户配置文件并随应用重启自动读取
- 代理开关默认关闭
- 用户可在不提供用户名与密码的情况下使用代理

## 用户场景与验收
1. 用户进入网络代理配置区域，开启代理开关并填写必要信息，保存后提示成功
2. 用户关闭代理开关并保存，应用后续网络访问不使用代理
3. 用户配置 SOCKS4/SOCKS5 代理并保存，功能与 HTTP/HTTPS 一致
4. 用户重启应用，之前保存的代理配置自动回填并保持状态一致

## 功能需求
- FR-1：系统提供全局代理开关，支持开启与关闭
- FR-2：系统支持四种代理类型：HTTP、HTTPS、SOCKS4、SOCKS5
- FR-3：系统支持配置代理地址与端口
- FR-4：系统支持可选的代理用户名与密码
- FR-5：代理配置保存到现有配置文件并可在重启后自动读取
- FR-6：新增代理配置区域与现有界面风格一致，不改变原有功能入口

## 成功标准
- 用户在 3 分钟内完成代理配置并保存成功
- 代理开启后，依赖外部网络访问的功能可在受限网络环境下正常完成
- 重启应用后，代理配置与开关状态保持一致且无需再次设置
- 现有功能在开启或关闭代理时均无行为回归

## 关键实体
- 代理配置：开关、类型、地址、端口、用户名、密码

## UI 与交互
- 在现有“网络代理”相关页面内新增配置区域
- 表单字段与按钮风格与当前 UI 组件一致
- 保存与失败提示沿用现有反馈风格
