# 项目宪章

## 元信息

- 项目名称: MTGA
- 宪章版本: 0.1.0
- 通过日期: 2026-02-01
- 最近修订: 2026-02-01

## 核心原则

### 原则 1: 功能不回归
- 必须在新增网络代理模块的同时保持现有功能不变
- 不改变原有“运行时选项”“代理服务”“网络检查”等行为与入口

### 原则 2: 配置单源与一致性
- 网络代理配置必须写入现有用户配置文件
- 应用重启后必须自动读取并回填配置
- 禁止新增独立配置文件用于代理

### 原则 3: 代理协议与鉴权覆盖
- 必须支持 http、https、socks4、socks5 四类代理
- 代理用户名与密码为可选项
- 代理开关为全局总开关，默认关闭

### 原则 4: 安全与隐私
- 不在日志中输出代理密码
- 仅在必要的网络请求链路中使用代理配置

### 原则 5: 影响范围最小化
- 仅调整出站网络请求的代理行为
- 不改变业务逻辑与数据模型的语义含义

## 治理

### 修订流程
- 修改宪章必须记录版本号与修订日期
- 任何偏离“核心原则”的变更必须在修订说明中明确原因

### 版本策略
- 重大原则变更或删除: 主版本号递增
- 新增原则或治理条款: 次版本号递增
- 文案澄清与非语义修订: 修订号递增

### 合规审查
- 每次实施需求前，应对照“核心原则”检查是否满足
- 若模板缺失，应补齐后再作为强制流程执行
